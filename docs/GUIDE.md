# 言枢使用指南

本文覆盖应用组装、数据响应、安全中间件、静态文件、OpenAPI 和无端口测试。全部示例对应当前源码；完整签名见[API 参考](API_REFERENCE.md)。

## 1. 建立项目

```text
my-service/
├── src/主.yx
├── templates/home.yb.html
├── static/app.css
└── 言序.toml
```

```sh
yanbao add web --version '^0.2'
```

应用代码通常把包引为`言枢`：

```yanxu
引「包:web」为 言枢；
```

## 2. 配置与资源预算

```yanxu
定 配置项 为 言枢.配置（）
    .设应用名（「用户服务」）
    .设模板根（「templates」）
    .设静态根（「static」）
    .设自动安全首部（真）
    .设内容安全策略（「default-src 'self'」）
    .设最大请求正文字节（1048576）
    .设最大响应正文字节（2097152）
    .设最大路由（1024）
    .设最大中间件（64）；

定 应用 为 言枢.创建应用（配置项）；
```

`创建应用`会按配置注册默认安全首部中间件。配置必须在启动前完成；运行中替换模板根、静态根或安全策略会让部署行为难以审计。

## 3. 路由与反向 URL

```yanxu
法 文章详情（上下文） 则
    归 言枢.JSON响应（{
        「编号」：上下文.参数值（「id」），
        「预览」：上下文.查询值（「preview」）
    }）；
终

应用.命名取（「/posts/:id」，「文章详情」，文章详情）；

定 地址 为 应用.反向（「文章详情」，{「id」：「first post」}）；
# /posts/first%20post
```

模式支持字面段、`:名称`单段参数和只位于末尾的`*名称`剩余路径。路由按注册顺序匹配；重复方法与模式会失败，不会覆盖旧处理器。

路由组只是注册期组合，不创建隐藏路由器：

```yanxu
定 接口 为 应用.命名路由组（「/api/v1」，「api」）；
接口.命名取（「/posts/:id」，「文章详情」，文章详情）；

定 地址 为 应用.反向（「api:文章详情」，{「id」：「42」}）；
```

## 4. 中间件与错误边界

中间件接收`上下文`和`下一步`，按注册顺序进入、逆序退出：

```yanxu
法 请求编号中间件（上下文，下一步） 则
    置 上下文.状态【「请求编号」】 为 「req-42」；
    定 响应 为 下一步（）；
    响应.设首部（「x-request-id」，「req-42」）；
    归 响应；
终

应用.使用（请求编号中间件）；
```

404 与 405 可分别替换；405 响应的`Allow`由路由结果生成。统一错误处理器接收上下文和错误，必须返回`HTTP响应`。处理器、中间件或错误处理器返回其他类型时，框架返回稳定的响应类型错误。

```yanxu
定 详情 为 言枢.错误详情（所误）；
若 （详情【「代码」】 等于 「YANSHU_LIMIT_REQUEST_BODY」） 则
    # 按稳定代码分支
终
```

## 5. JSON、言据与内容协商

JSON 请求必须带 JSON 媒体类型：

```yanxu
法 创建JSON（上下文） 则
    定 输入：典 为 上下文.JSON正文（）；
    归 言枢.JSON响应（{「姓名」：输入【「姓名」】}）；
终
```

言据正文只接受言据媒体类型，并通过`yanju`的受限解析器：

```yanxu
法 创建言据（上下文） 则
    定 输入：典 为 上下文.言据正文（）；
    归 言枢.言据值响应（输入）；
终
```

`言据响应（正文）`接收已经规范序列化的文字；`言据值响应（值）`负责序列化言序值。不要把 JSON 文本传给言据响应。

`协商响应（上下文，值）`解析`Accept`中的质量与特异度，在 JSON 和言据间选择，并写入`Vary: accept`。两种媒体类型均不可接受时返回`YANSHU_NOT_ACCEPTABLE`。

## 6. 会话

会话存储是显式协议：

```yanxu
公 类 数据库存储 纳 言枢.会话存储协议 则
    公 法 读取（编号：文）：典? 则
        # 从数据库读取隔离数据
    终
    公 法 写入（编号：文，数据：典，最大秒：数）：空 则
        # 原子写入并设置过期
    终
    公 法 删除（编号：文）：空 则
        # 幂等删除
    终
终
```

也可包装三个回调：

```yanxu
定 存储 为 言枢.会话存储适配器（读取法，写入法，删除法）；
```

会话配置示例：

```yanxu
定 会话项 为 言枢.会话配置（）
    .设Cookie名（「__Host-yanxu-session」）
    .设Cookie路径（「/」）
    .设最大秒（3600）
    .设最大数据字符（32768）
    .设安全（真）
    .设仅HTTP（真）
    .设同站（「Lax」）；

应用.使用（言枢.会话中间件（会话项，存储））；
```

处理器通过`上下文.会话（）`读取会话对象，再调用`取/设/删值/轮换/销毁/所有/须保存/须轮换/已销毁`。存储和快照都做深复制；外部修改不会回写内部状态。生产存储必须自行提供并发控制、过期清理、容量治理和可观测性。

## 7. CSRF

CSRF 中间件默认保护非安全方法。推荐配置明确的 HTTPS 来源：

```yanxu
定 防护 为 言枢.CSRF配置（）
    .加可信来源（「https://app.example.com」）
    .设令牌首部名（「x-csrf-token」）
    .设令牌Cookie名（「__Host-yanxu-csrf」）
    .设安全（真）
    .设仅HTTP（假）
    .设同站（「Strict」）；

应用.使用（言枢.CSRF中间件（防护））；
```

浏览器先获得由`CSRF令牌（）`创建的令牌及`CSRF令牌Cookie`，提交时把同一令牌写入配置首部。中间件同时检查`Sec-Fetch-Site`、`Origin`或`Referer`，再比较首部与 Cookie。认证、授权和 HTTPS 仍是独立必需边界。

## 8. 静态文件

```yanxu
定 静态项 为 言枢.静态配置（）
    .设最大文件字节（4194304）
    .设缓存控制（「public, max-age=60」）
    .设条件请求（真）
    .设范围请求（真）
    .设允许隐藏文件（假）；

应用.挂配置静态为（「/assets」，静态项）；
```

静态边界拒绝绝对路径、反斜杠、NUL、`.`、`..`和默认隐藏文件。响应支持`If-None-Match`、`If-Modified-Since`、单一`bytes`范围和`If-Range`；多范围不会伪装为完整 multipart 实现。ETag 是基于文件状态的弱验证器，不是内容摘要。

## 9. OpenAPI

显式说明和路由登记：

```yanxu
定 说明 为 言枢.接口说明（「users.get」）
    .设摘要（「读取用户」）
    .设说明（「按编号读取单个用户」）
    .设标签（【「用户」】）
    .设参数（【{「名称」：「id」，「位置」：「path」，「必需」：真}】）
    .设响应（{「200」：{「说明」：「成功」}}）
    .设扩展（「x-owner」，「identity」）；

应用.命名接口路由（「GET」，「/users/:id」，「用户详情」，说明，读取用户）；
```

提供器实现`OpenAPI提供器协议.生成（接口快照）`。回调入口适合小型项目：

```yanxu
定 提供器 为 言枢.OpenAPI提供器（生成文档）；
应用.挂OpenAPI（「/openapi.json」，提供器）；
```

操作标识全局唯一；普通路由不进入快照。元数据、快照、提供器输入和输出都隔离复制。单操作预算为 65536 字符，应用快照和生成文档各为 1 MiB。

## 10. 无端口测试客户端

```yanxu
定 客户端 为 言枢.测试客户端（应用）；

定 响应 为 客户端.请求字节为（
    「PATCH」，
    「/users/42?q=active」，
    {「Content-Type」：「application/octet-stream」，「X-Trace-ID」：「trace-1」}，
    数据字节
）；
```

`请求为`发送任意首部，`请求字节为`发送原始字节正文，`请求Cookie`从 Cookie 典生成请求。测试客户端自行管理`Content-Length`，拒绝调用方注入`Content-Length`、`Transfer-Encoding`或`Trailer`。

## 11. 言访测试传输

```yanxu
引「包:web/言访测试」为 言访测试；

定 客户端 为 言访测试.客户端（应用）；
定 响应 为 客户端.发文（「/users」）
    .言据正文（{「姓名」：「子衿」}）
    .发送（）；
```

适配器保留请求查询、首部、Cookie、字节正文、受控重定向、重复响应首部和 HEAD 语义。它不打开网络，也不模拟 TLS、DNS、连接失败或真实超时。所有 HTTP(S) 地址都会映射到传入的同一个应用。

## 12. 言标与 HTML

模板文件使用`.yb.html`扩展名。普通`{{ 值 }}`自动 HTML 转义；只有`言枢.言标.信任HTML（内容）`创建的值可以绕过。可信包装是审计标记，不是清洗器。完整语法见[言标参考](yanbiao.md)。

## 13. 运行示例与基准

从多仓工作区根执行：

```sh
/tmp/yanxu-v1.1.12/target/release/yanxu yanxu-libraries-workspace/repos/yanxu-web/examples/无端口言访.yx
/tmp/yanxu-v1.1.12/target/release/yanxu yanxu-libraries-workspace/repos/yanxu-web/benchmarks/路由与言访.yx -- 20
```

开发服务器示例会监听回环地址；无端口示例和基准不申请监听权限。
