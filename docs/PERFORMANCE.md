# 性能与资源预算

## 复杂度

| 路径 | 主要复杂度 | 分配 |
| --- | --- | --- |
| 路由登记 | 与路由数线性检查重复 | 模式段与名称索引 |
| 路由解析 | 与路由数和路径段数相关 | 路径参数典 |
| 中间件 | 与中间件数线性 | 每层下一步闭包 |
| JSON/言据正文 | 与正文字节线性 | 完整解析值 |
| 会话读写 | 与会话数据大小线性 | 深复制与规范大小检查 |
| CSRF | 与来源和令牌长度线性 | 小型临时文字 |
| 静态文件 | 与文件字节线性 | 当前实现完整读取文件 |
| 接口快照 | 与显式接口元数据总量线性 | 深复制与规范序列化 |
| 言访适配 | 与首部项和正文字节线性 | 请求对象、响应首部列和正文引用 |

路由器是有序线性匹配器，不声称常数时间。开发服务器串行处理连接，不是吞吐型生产宿主。

## 默认预算

- 请求正文：4 MiB；
- 响应正文：8 MiB，可配置上限 16 MiB；
- 路由：4096；
- 中间件：256；
- 显式 OpenAPI 接口：512；
- 单操作 OpenAPI 元数据：65536 字符；
- 接口快照：1 MiB；
- OpenAPI 文档：1 MiB；
- 会话和 CSRF 预算由对应配置进一步收紧；
- 静态首部、Cookie、路径和范围继续受`yanxu-http`边界限制。

预算是拒绝服务防线，不是容量目标。部署应根据请求模型设置更小值，并在反向代理和应用层同时限制。

## 基准

`benchmarks`是锁定言枢 1.x 路径依赖的独立消费者包。`benchmarks/路由与言访.yx`比较同一应用上的两条确定性路径：

1. `言枢测试客户端`直接请求；
2. 言访请求构建器经`言访测试`适配器请求。

两条路径都解析 JSON 并校验路由参数。运行：

```sh
/tmp/yanxu-v1.1.12/target/release/yanxu \
  yanxu-libraries-workspace/repos/yanxu-web/benchmarks/路由与言访.yx -- 10
```

输出字段：

```text
轮数
直接毫秒
直接校验
言访毫秒
言访校验
```

`直接校验`和`言访校验`必须都等于轮数，否则基准失败。毫秒结果受硬件、解释器模式、缓存和系统负载影响，不应跨机器直接比较。

## 测量建议

- 固定言序版本、依赖锁和轮数；
- 先运行一次预热；
- 同一进程内比较前后实现；
- 分别报告树解释器和已构建 VM；
- 同时记录输入大小、路由数、中间件数和响应大小；
- 不把无端口适配器结果当作真实网络延迟。

## 优化边界

优先优化可测量的热路径，保持安全检查和错误分类。禁止通过跳过正文上限、路径验证、深复制、Cookie 校验或失败原子性换取基准数字。

高并发生产需求应使用成熟宿主提供连接调度，并在真实负载下测量；内建开发服务器不应作为生产吞吐基准。
