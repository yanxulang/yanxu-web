# 言枢公开 API

当前公开版本为`0.2.0`；技术包标识为`yanxu-web`。

## 模块工厂

| API | 作用 |
| --- | --- |
| `配置（）` | 新建`言枢配置`。 |
| `创建应用（配置）` | 新建配置化应用，并按配置加入默认安全首部。 |
| `应用（）` | 0.1 兼容工厂，不自动注册安全首部。 |
| `服务器（应用，地址）` | 新建`言枢开发服务器`。 |
| `开发服务器（应用，地址）` | 0.1 兼容服务器工厂。 |
| `测试客户端（应用）` | 新建无端口请求客户端。 |

## `言枢配置`

字段为`应用名/调试/模板根/静态根/自动安全首部/内容安全策略`。链式设置法为`设应用名/设调试/设模板根/设静态根/设自动安全首部/设内容安全策略`。

`调试`当前只作为应用级配置标记，不会把错误细节发送给客户端。

## `言枢应用`

### 路由

- `路由（方法，模式，处理器）`
- `命名路由（方法，模式，名称，处理器）`
- `取/发/更新/修补/删（模式，处理器）`
- `命名取/命名发（模式，名称，处理器）`
- `路由组（前缀）`
- `命名路由组（前缀，命名空间）`
- `反向（名称，参数）`

模式支持字面段、`:名称`单段参数和只位于末尾的`*名称`剩余路径参数。反向 URL 会百分号编码单段参数；通配参数按斜线分段编码。

### 生命周期与响应

- `使用（中间件）`
- `设错误处理器（处理器）`
- `设状态处理器（404|405，处理器）`
- `处理（HTTP请求）`
- `模板响应（模板名，数据）`
- `模板状态响应（状态码，模板名，数据）`
- `挂静态（URL前缀，文件根）`
- `挂配置静态（URL前缀）`

处理器、中间件和错误处理器必须返回`HTTP响应`，否则进入稳定的响应类型错误。

## `言枢上下文`

| API | 结果 |
| --- | --- |
| `参数值（名称）` | 路径参数或空。 |
| `查询值（名称）` | 第一项查询值或空。 |
| `查询全部（名称）` | 全部查询值。 |
| `首部值（名称）` | 请求首部或空。 |
| `Cookie值（名称）` | Cookie 或空。 |
| `正文文字（）` | 严格 UTF-8 正文。 |
| `JSON正文（）` | 校验 JSON 内容类型后解析正文。 |
| `是AJAX（）` | 判断`X-Requested-With: XMLHttpRequest`。 |
| `状态值（名称）` | 应用共享状态值或空。 |
| `反向（名称，参数）` | 由绑定应用生成 URL。 |

兼容类型`Web上下文`继承`言枢上下文`，现有类型标注无需立即迁移。

## 响应工厂

- `HTML响应/HTML状态响应`：渲染`yanxu-html`节点；
- `言标源码响应/言标响应/言标状态响应`：渲染言标源码或文件；
- `JSON响应/言据响应/文字响应/字节响应/状态响应`；
- `重定向/永久重定向/空响应/Cookie`。

`言标响应`接收模板环境；应用代码通常优先使用`应用.模板响应`。

## `言枢.言标`

- `环境（模板根）`
- `渲染（源码，数据）`
- `信任HTML（内容）`

文件环境提供`渲染源码`与`渲染文件`。语法与安全约束见[言标参考](yanbiao.md)。

## `言枢测试客户端`

- `请求（方法，路径）`
- `取（路径）`
- `带正文（方法，路径，正文，内容类型）`
- `发（路径，正文，内容类型）`
- `发JSON（路径，值）`
- `删（路径）`

测试客户端直接调用`应用.处理`，覆盖路由、中间件、模板与错误边界，不覆盖真实套接字读写；真实连接只需保留少量冒烟测试。
