# 言枢架构

## 分层与依赖方向

```text
业务处理器 / 中间件 / OpenAPI 提供器
                    │
                    ▼
       言枢应用 · 路由 · 上下文 · 测试边界
          │          │          │
          ▼          ▼          ▼
       言标/言页    言据      言访测试传输
          │          │          │
          └──────┬───┴──────────┘
                 ▼
              yanxu-http
```

`yanxu-http`拥有 HTTP/1.1 解析、首部、Cookie、媒体类型、表单和响应编码；言枢不复制协议实现。`yanxu-html`拥有安全节点与 URL/名称规则；言标复用其转义边界。`yanju`拥有言据解析、规范序列化、限制和深复制。`yanxu-request`不被默认框架入口导入，只由独立`言访测试`导出模块用于无端口客户端互通。

依赖方向保持单向：言访不依赖言枢，言枢通过言访的公开自定义传输协议建立测试适配器，因此不存在客户端与服务端循环依赖。

## 应用组装

`创建应用（配置）`执行以下操作：

1. 创建独立路由器、状态典、中间件列和显式接口登记列；
2. 按路由与中间件预算配置容器；
3. 建立绑定模板根的言标环境；
4. 在启用时注册默认安全首部中间件。

应用不使用全局可变路由表或全局会话。多个应用实例可以在同一进程中使用不同配置、存储、模板根和安全策略。

## 请求生命周期

1. 开发服务器从套接字读取受首部、正文和超时预算约束的`HTTP请求`；测试客户端直接构造等价对象。
2. 应用创建`言枢上下文`，绑定请求、路径参数、应用状态、反向路由法和可选会话。
3. 路由器按注册顺序解析方法和路径，返回处理器、参数、允许方法与 404/405 状态。
4. 中间件链按注册顺序进入；`下一步`每层只能推进一次，任一层可短路。
5. 端点返回`HTTP响应`。框架验证响应类型和正文预算。
6. 任意错误进入应用错误处理器；错误处理器的返回值再次接受类型验证。
7. 开发服务器按请求方法编码响应并关闭连接；内存测试边界直接返回响应对象。

404 与 405 不是异常路径。405 使用匹配同一路径的允许方法生成`Allow`；500 只处理执行失败或违反框架契约的情况。

## 路由模型

路由模式在注册时解析为有序段：

- 字面段精确匹配；
- `:名称`匹配一个百分号解码后的段；
- `*名称`只允许位于末尾并匹配剩余路径。

路由表是有序列，解析复杂度与已登记路由数和路径段数相关。重复方法与模式在写入前拒绝。命名路由维护独立名称索引；反向 URL 根据同一模式生成，并对动态段做百分号编码。

显式 OpenAPI 登记先验证操作标识，再写入真实路由器，最后追加隔离元数据。重复操作标识不会留下路由；重复路由不会污染接口快照。

## 中间件链

中间件链持有当前下标、上下文和最终端点。调用`下一步`会创建后续闭包并进入下一层；退出顺序自然反转。这个模型不依赖线程局部变量，事务、会话和追踪信息通过上下文或显式对象传递。

安全首部、会话与 CSRF 都是普通中间件。它们没有特殊调度权限，因此注册顺序会影响行为：通常先放请求追踪与错误观测，再放会话和 CSRF，最后放业务中间件。

## 会话状态机

会话中间件的边界如下：

```text
请求 Cookie
  → 校验编号
  → 存储读取隔离数据
  → 绑定言枢会话
  → 执行业务链
  → 根据 须保存/须轮换/已销毁 决定写入或删除
  → 输出 Set-Cookie
```

会话对象追踪原编号、当前编号、数据、变更、轮换和销毁状态。读取、设值、全部快照及存储调用都复制受支持数据，避免调用方通过共享典回写内部状态。数据字符预算在写入前再次检查。

存储协议只定义`读取/写入/删除`。连接池、数据库事务、跨进程锁、过期清扫和故障恢复属于具体存储实现，不由框架伪装完成。

## CSRF 状态机

中间件跳过 GET、HEAD、OPTIONS 和 TRACE。其他方法依次执行：

1. 拒绝明确的跨站`Sec-Fetch-Site`；
2. 在配置了可信来源时校验`Origin`，缺失时回退`Referer`；
3. 读取配置首部和 Cookie 中的令牌；
4. 验证格式、长度并做完整遍历比较；
5. 失败返回 403，成功推进业务链。

令牌 Cookie 默认`Secure`和`SameSite=Strict`。`__Host-`名称必须保持`Secure`、路径`/`且不能设置 Domain。CSRF 不提供用户身份或权限判定。

## 静态文件管线

静态处理器先解析通配路径，再执行路径隔离和隐藏文件策略，然后读取文件状态与字节。条件和范围处理顺序为：

1. 计算媒体类型、弱 ETag 和修改日期；
2. 处理`If-None-Match`，再处理`If-Modified-Since`；
3. 在启用范围时解析单一`Range`；
4. 用`If-Range`决定返回 206 还是完整 200；
5. 写入缓存、安全和范围首部。

416 会返回`Content-Range: bytes */总字节`。框架没有实现 multipart 多范围，因此不会把多个范围静默拼成错误响应。

## OpenAPI 边界

`接口操作`保存显式元数据。构建器输入在写入时复制，`快照`再生成独立结构。应用只保存已登记操作，不扫描普通路由或处理器源码。

`OpenAPI提供器协议`接收应用快照并返回文档典。回调包装器再次复制输入；应用对输出做类型、大小和隔离检查。HTTP 入口使用`application/vnd.oai.openapi+json; version=3.1; charset=utf-8`。

框架不验证提供器输出是否满足完整 OpenAPI Schema。这样可以替换生成器而不把某一规范实现绑定到路由核心；调用方应在 CI 中用专门 OpenAPI 校验器检查最终文档。

## 测试边界

`言枢测试客户端`通过`yanxu-http`解析器构造请求，因此自定义首部、Cookie、查询和正文遵守与套接字入口相同的协议验证。它管理正文长度并拒绝分帧首部注入。

`言访测试`模块把言访请求快照转换成 origin-form 请求，再把 HTTP 响应编码的首部区转换成有序首部列。这样可保留多个`Set-Cookie`，同时返回语义正文。HEAD 响应保留实体长度首部但清空客户端正文。

适配器是进程内函数调用，不覆盖 DNS、TLS、套接字超时、连接复用、服务端断开或代理行为。这些边界必须由少量真实回环集成测试覆盖。

## 言标渲染

模板加载先验证安全相对路径与`.yb.html`扩展名，再施加源码和包含深度预算。词法阶段区分静态 HTML、自动转义插值、命令与注释；渲染阶段使用父作用域链实现循环变量，不修改调用方顶层数据。

继承先收集子模板块，再把块结果交给基础模板。动态值在块渲染时仍自动转义。只有显式`信任HTML`实例绕过转义。

## 开发服务器

内建服务器用于开发、教学和回环集成：串行接受连接，每连接读取一个请求、写一个最终响应并关闭。它没有 TLS、长连接工作池、HTTP/2/3、流式请求、代理信任、生产背压、优雅重启或多进程协调。

生产架构应由成熟前置服务器处理 TLS、连接复用、慢客户端和代理边界，并把受信请求转给经过部署验证的应用宿主。
