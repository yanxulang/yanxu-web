# 言枢架构与请求生命周期

## 分层

```text
业务处理器 / 页面数据
          │
          ▼
言枢应用 ─ 路由 · 中间件 · 状态处理器 · 测试客户端
    │                         │
    ▼                         ▼
  言标                    静态文件服务
    │                         │
    └──────────┬──────────────┘
               ▼
        yanxu-html / yanxu-http
```

言标属于言枢发行与 API 面，但继续复用标准模板转义能力；`yanxu-html`保留节点模型、名称与 URL 安全规则；`yanxu-http`保留协议对象和字节编码。言枢不复制底层协议解析。

## 请求生命周期

1. 开发服务器从连接读取一个受预算约束的`HTTP请求`；测试客户端可直接构造等价请求。
2. 应用建立`Web上下文`兼容实例，它同时属于`言枢上下文`，绑定应用状态与反向路由法。
3. 路由器按注册顺序匹配路径，收集路径参数与允许方法。
4. 未匹配路径选择 404；路径匹配但方法不符选择 405，并写入`Allow`。
5. 中间件按注册顺序进入、逆序退出；任一中间件可以短路。
6. 处理器可返回言标页面、HTML 节点、JSON、言据、文字、字节或重定向响应。
7. 处理器和中间件错误交给统一错误处理器；框架再次验证最终值必须是`HTTP响应`。
8. 开发服务器编码并发送响应，然后关闭连接。

## 配置与组装

`言枢配置`在应用创建时生成言标环境。配置对象之后仍可读取，但应在服务启动前完成设置，避免运行中改变模板根或安全策略。

命名路由组只是注册期组织工具，不引入隐藏的第二套路由器。组内模式与前缀合并后进入同一有序路由表，命名空间用`命名空间:名称`生成稳定名称。

## 言标渲染

模板加载先验证安全相对路径与`.yb.html`扩展名，再施加 256 KiB 源码预算、16 层包含预算和循环检测。词法阶段区分静态文字、自动转义插值、中文命令和注释；渲染阶段以父作用域链实现循环变量，不修改调用方的顶层数据典。

继承时，子模板先渲染命名块，再把结果交给基础模板对应块位置。动态插值仍在块渲染时自动转义。多层继承保留最下层的同名覆盖。

## 安全不变量

- 普通言标插值与`yanxu-html`文字节点默认转义；
- 可信 HTML 必须经名称明确的包装创建；
- 反向 URL 编码动态路径参数；
- 静态文件与模板文件都拒绝绝对路径、反斜杠、NUL、`.`和`..`；
- 默认安全中间件不泄漏错误详情；
- 返回类型错误不会被静默转换为成功响应。

应用仍负责认证、授权、CSRF 策略、领域校验、速率限制、密钥、日志脱敏与部署信任边界。

## 当前服务器边界

0.2 的服务器仍是本地开发服务器：HTTP/1.1、串行接受、一连接一请求、`Connection: close`。它没有 TLS、并发工作池、长连接、请求流、代理信任、访问日志、指标、优雅关闭或生产背压。

生产化应由成熟前置服务器处理 HTTPS 和静态资产，并等待底层 HTTP 与异步运行时提供经过资源预算和互操作验证的并发模型。
