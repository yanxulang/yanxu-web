# 从 yanxu-web 0.1 迁移到言枢 0.2

0.2 保留技术包标识`yanxu-web`和主要 0.1 API，因此可以分阶段迁移。

## 1. 更新模块别名

```yanxu
# 旧
引「包:yanxu-web」为 Web；

# 新
引「包:yanxu-web」为 言枢；
```

这一步只改变源码可读性，不改变包解析。

## 2. 引入配置化创建

```yanxu
# 旧
定 应用 为 言枢.应用（）；

# 新
定 配置项 为 言枢.配置（）
    .设模板根（「templates」）
    .设静态根（「static」）；
定 应用 为 言枢.创建应用（配置项）；
```

`创建应用`默认注册安全首部。若旧项目已有同类中间件，应删除重复实现，或先`设自动安全首部（假）`。

## 3. 命名路由

```yanxu
应用.命名取（「/posts/:id」，「文章详情」，文章）；
定 地址 为 上下文.反向（「文章详情」，{「id」：「first」}）；
```

可以先为会被页面链接的路由命名，再逐步消除硬编码 URL。

## 4. 从 HTML 节点迁移到言标

`HTML响应`继续支持`yanxu-html`节点，不要求一次性重写。新页面可放入`.yb.html`文件并改用：

```yanxu
归 应用.模板响应（「post.yb.html」，{「文章」：文章}）；
```

普通动态值自动转义。原有`HTML.原始`使用点迁移时必须逐项审计；只有确认可信的数据才改为`言枢.言标.信任HTML`。

## 5. 测试客户端

旧测试可继续构造`HTTP请求`并调用`应用.处理`。推荐改为：

```yanxu
定 客户端 为 言枢.测试客户端（应用）；
定 响应 为 客户端.取（「/posts/first」）；
```

需要覆盖套接字边界时再保留少量真实连接冒烟测试。

## 6. 类型名

`Web应用`继承`言枢应用`，`Web上下文`继承`言枢上下文`，`Web开发服务器`继承`言枢开发服务器`。0.1 类型标注可继续工作；新公开接口建议使用言枢类型名。

## 7. 依赖发布顺序

跨仓升级必须先发布言枢 0.2，再把消费项目的版本要求更新为`^0.2`并重新生成`言序.lock`。不要手工猜测 Git 修订或内容校验和。
