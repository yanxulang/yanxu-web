# 安全模型

## 信任边界

| 输入 | 默认信任 | 处理 |
| --- | --- | --- |
| 请求行、首部、Cookie、查询、正文 | 不可信 | `yanxu-http`严格解析与预算 |
| JSON/言据正文 | 不可信 | 媒体类型验证与受限解析 |
| 路径参数 | 不可信 | 百分号解码后交给业务验证 |
| 模板数据 | 不可信 | 普通插值自动 HTML 转义 |
| 可信 HTML | 高信任 | 只经显式包装，不做清洗 |
| 会话存储返回值 | 半可信 | 类型、大小和深复制边界 |
| OpenAPI 提供器输出 | 半可信 | 类型、大小和复制；不做完整 Schema 校验 |
| 静态文件路径 | 不可信 | 根目录约束与危险段拒绝 |

业务应用仍负责认证、授权、领域校验、速率限制、密钥管理、审计日志和隐私治理。

## 请求与响应预算

应用在进入路由前检查请求正文上限，在中间件和处理器完成后检查响应正文上限。路由、中间件、会话数据、CSRF 配置、静态响应和 OpenAPI 元数据还具有各自预算。

底层 HTTP 解析拒绝冲突分帧、非法首部、重复 Host/Content-Length、危险 Cookie 和超限字段。测试客户端也走同一解析器；`Content-Length`、`Transfer-Encoding`和`Trailer`由测试边界管理，调用方不能注入。

## 错误暴露

默认错误响应不包含内部异常消息。`错误详情`只供受信代码分类，日志也应在写出前脱敏。不得记录：

- Authorization；
- Cookie 与 Set-Cookie；
- CSRF 令牌；
- 会话编号和完整会话数据；
- 数据库、API 或签名密钥；
- 未经脱敏的个人字段。

## 会话

安全默认值使用`__Host-yanxu-session`、Secure、HttpOnly、路径`/`和 SameSite=Lax。`__Host-`约束在配置时验证。

会话数据在存储输入、读取、对象访问和快照处复制。编号轮换会删除旧编号并写入新编号；销毁会删除存储并输出过期 Cookie。存储实现必须提供：

- 原子写入；
- 并发更新策略；
- 过期清扫；
- 容量和失败监控；
- 静态加密或等价宿主保护；
- 不泄漏编号的日志策略。

框架不提供进程间锁、数据库或分布式会话存储。

## CSRF

CSRF 中间件保护非安全方法。默认模型是：

1. 检查 Fetch Metadata；
2. 在配置时检查 Origin，缺失时回退 Referer；
3. 验证首部令牌和 Cookie 令牌的格式与预算；
4. 完整遍历比较两个令牌；
5. 失败返回 403。

双提交令牌必须使用不可预测生成器并通过 HTTPS 传输。它不能替代登录态、权限校验或防重放业务令牌。若应用允许跨站 API、无浏览器客户端或多源部署，应明确配置并单独审计。

## 静态文件

静态路径拒绝绝对路径、反斜杠、NUL、空段、`.`、`..`和默认隐藏文件。根目录与候选路径经过规范化约束。返回值默认带`X-Content-Type-Options: nosniff`。

ETag 是弱验证器，适合缓存协商，不用于完整性或授权。范围请求只支持单范围，避免不完整 multipart 实现。静态处理器不会进行用户级授权；私有文件应由业务处理器校验后读取。

## 言标与 HTML

普通插值自动 HTML 转义。`信任HTML`是显式信任升级标记，不是 HTML 清洗器。不得把用户输入、数据库富文本、Markdown 原始 HTML 或第三方响应直接包装为可信 HTML。

CSP 默认值是基线，不是适合所有应用的最终策略。加入脚本、样式、图片、iframe 或外部资源时应最小化来源并验证浏览器行为。

## OpenAPI

只有显式接口登记进入快照，避免意外公开内部路由。操作标识唯一，扩展名必须以`x-`开头，元数据和文档受大小限制并深复制。

提供器仍可能生成语义错误或包含敏感描述。发布 OpenAPI 前应：

- 用完整 OpenAPI 3.1 校验器检查；
- 审查示例、默认值和扩展；
- 不暴露内部主机、错误堆栈、密钥或测试凭据；
- 按部署权限决定文档端点是否公开。

## 言访测试传输

适配器不打开套接字，但会把任何 HTTP(S) 地址映射到同一应用。测试不能据此证明域名隔离、TLS、DNS、证书、代理或超时正确。

跨域重定向的敏感首部移除由言访客户端执行；适配器只负责把每一跳送入应用并保留响应首部。真实网络安全仍需回环或隔离环境集成测试。

## 开发服务器

内建服务器没有生产级 TLS、并发、慢连接防护、代理信任、访问日志、指标、优雅重启和多进程协调。不要直接暴露到公网。

推荐部署边界：

```text
公网
  → 成熟反向代理（TLS、连接与速率限制）
  → 受控应用宿主
  → 言枢应用
  → 显式会话/数据存储
```

仅在明确配置并验证时信任代理转发首部。
