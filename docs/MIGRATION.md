# 迁移到言枢 1.0

本文面向`yanxu-web 0.1/0.2`用户。1.0 保留主要兼容类型与工厂，但收紧资源、安全和测试边界，并增加显式集成能力。

## 1. 工具链与依赖

最低言序版本提高到`1.1.12`。先更新工具链，再重新解析锁文件：

```sh
yanbao install --manifest-path 项目
yanbao check --manifest-path 项目
yanbao test --manifest-path 项目 -- --并发 1
```

1.0 稳定线依赖`yanxu-http ^1.0`、`yanxu-html ^1.0`、`yanju ^1.2`和`yanxu-request ^1.0`。不要把临时分支或本地 Git 提交写入发布清单。

## 2. 模块别名与兼容类型

技术包名仍是`yanxu-web`。推荐把默认模块引为`言枢`：

```yanxu
# 旧写法仍可工作
引「包:yanxu-web」为 Web；

# 推荐
引「包:yanxu-web」为 言枢；
```

`Web应用`继承`言枢应用`，`Web上下文`继承`言枢上下文`，`Web开发服务器`继承`言枢开发服务器`。新公共签名应使用言枢类型名；旧类型标注可分阶段迁移。

## 3. 配置化创建

旧代码：

```yanxu
定 应用 为 言枢.应用（）；
```

推荐代码：

```yanxu
定 配置项 为 言枢.配置（）
    .设模板根（「templates」）
    .设静态根（「static」）
    .设最大请求正文字节（1048576）
    .设最大响应正文字节（2097152）；

定 应用 为 言枢.创建应用（配置项）；
```

`创建应用`默认加入安全首部。旧项目已有同类中间件时，应删除重复实现，或显式`设自动安全首部（假）`。

## 4. 正文与内容协商

JSON 继续使用`JSON正文/JSON响应`。言据新增严格媒体类型和受限解析：

```yanxu
法 创建（上下文） 则
    定 输入 为 上下文.言据正文（）；
    归 言枢.言据值响应（输入）；
终
```

旧`言据响应`接收规范文本，不接收任意典。传入值时必须改用`言据值响应`。需要在 JSON 与言据间协商时使用`协商响应`，并确保缓存尊重`Vary: accept`。

## 5. 会话改为显式存储

1.0 不提供进程全局会话字典。实现`会话存储协议`或包装三个回调：

```yanxu
定 存储 为 言枢.会话存储适配器（读取，写入，删除）；
应用.使用（言枢.会话中间件（言枢.会话配置（），存储））；
```

迁移时确认：

1. 读取返回空或受支持数据典；
2. 写入是原子的，并遵守最大秒；
3. 删除幂等；
4. 日志不记录会话编号、Cookie 或完整数据；
5. 多进程部署使用共享存储和明确并发策略。

`__Host-`会话 Cookie 必须保持`Secure`且路径为`/`。本地 HTTP 开发如需关闭 Secure，应改用不带`__Host-`的名称。

## 6. CSRF

状态变更路由应注册 CSRF 中间件：

```yanxu
定 防护 为 言枢.CSRF配置（）
    .加可信来源（「https://app.example.com」）；
应用.使用（言枢.CSRF中间件（防护））；
```

客户端必须同时发送令牌 Cookie 和配置首部。原先只检查自定义首部、只检查`Origin`或只依赖 SameSite 的实现应移除，避免出现两套冲突策略。

## 7. 静态文件

旧`挂静态`仍可用。要启用缓存与范围边界，改用：

```yanxu
定 选项 为 言枢.静态配置（）
    .设最大文件字节（4194304）
    .设缓存控制（「public, max-age=60」）
    .设条件请求（真）
    .设范围请求（真）；

应用.挂静态为（「/assets」，「static」，选项）；
```

依赖旧行为的测试需要更新：有效条件请求会返回 304，范围请求会返回 206，不满足范围返回 416。隐藏文件默认拒绝。

## 8. OpenAPI 显式登记

普通`取/发/路由`不会自动进入 OpenAPI 快照。需要文档的路由改为：

```yanxu
定 说明 为 言枢.接口说明（「users.get」）
    .设摘要（「读取用户」）
    .设响应（{「200」：{「说明」：「成功」}}）；

应用.接口取（「/users/:id」，说明，读取用户）；
```

操作标识必须唯一且符合稳定语法。提供器负责将快照转换为 OpenAPI 3.1 文档；框架不推断参数 Schema。

## 9. 测试客户端

旧便捷入口保留：

```yanxu
定 响应 为 言枢.测试客户端（应用）.取（「/」）；
```

自定义首部和字节正文使用：

```yanxu
定 响应 为 客户端.请求字节为（方法，路径，首部，正文）；
```

测试客户端现在管理`Content-Length`并拒绝`Transfer-Encoding/Trailer`注入。旧测试手工提供这些首部时应删除它们。

## 10. 言访集成

原先用 Mock 传输伪造服务端响应的测试，可改为真实驱动言枢应用：

```yanxu
引「包:yanxu-web/言访测试」为 言访测试；

定 客户端 为 言访测试.客户端（应用）；
定 响应 为 客户端.取（「/health」）.发送（）；
```

这个边界验证言访构建器、言枢路由、中间件、正文解析和响应模型，但不替代真实套接字/TLS 冒烟测试。

## 11. 文档路径

统一文档文件名改为大写稳定入口：

- `docs/API.md`；
- `docs/GUIDE.md`；
- `docs/ARCHITECTURE.md`；
- `docs/MIGRATION.md`；
- `docs/PERFORMANCE.md`；
- `docs/SECURITY_MODEL.md`。

更新仓库内或外部链接时使用这些路径。

## 12. 回归清单

迁移后至少验证：

- 404 与 405 及`Allow`；
- 请求/响应正文上限；
- JSON 和言据媒体类型；
- 会话新建、轮换、销毁和存储失败；
- CSRF 同源、缺失令牌和不匹配令牌；
- 静态 200/206/304/416；
- OpenAPI 重复标识与快照隔离；
- 无端口测试首部、Cookie、字节正文；
- 言访 JSON、言据、重定向、重复 Cookie 和 HEAD。
