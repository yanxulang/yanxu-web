# 言枢公开 API

稳定入口是`包:web`或清单别名对应的默认导出；言标入口是`包:web/言标`，言访适配器入口是`包:web/言访测试`。当前稳定版本为`1.0.0`，最低言序版本为`1.1.12`。

本文解释公共契约和错误边界。由编译器机器清单生成的全部字段、方法与精确签名见[API_REFERENCE.md](API_REFERENCE.md)。

## 模块常量与错误

| API | 含义 |
| --- | --- |
| `版本` | 框架源码版本。 |
| `名称` | 产品名`言枢`。 |
| `包标识` | 技术包名`yanxu-web`。 |
| `默认最大请求正文字节` | 默认 4 MiB。 |
| `默认最大响应正文字节` | 默认 8 MiB。 |
| `默认最大路由` | 默认 4096。 |
| `默认最大中间件` | 默认 256。 |
| `错误详情（错误）` | 返回稳定`代码/消息`典。 |

错误代码取第一个全角或半角冒号前的前缀。程序逻辑应判断代码，不应匹配消息。

## 配置

`配置（）`返回`言枢配置`。配置字段及对应链式设置法包括：

- `应用名/设应用名`；
- `调试/设调试`；
- `模板根/设模板根`；
- `静态根/设静态根`；
- `自动安全首部/设自动安全首部`；
- `内容安全策略/设内容安全策略`；
- `最大请求正文字节/设最大请求正文字节`；
- `最大响应正文字节/设最大响应正文字节`；
- `最大路由/设最大路由`；
- `最大中间件/设最大中间件`。

所有数值设置在写入时验证为受支持范围内的整数。`创建应用（配置）`创建现代应用；`应用（）`保留旧版兼容工厂，不自动组装安全首部。

## 应用与路由

`言枢应用`的路由入口：

| API | 契约 |
| --- | --- |
| `路由（方法，模式，处理器）` | 登记任意合法 HTTP 方法。 |
| `命名路由（方法，模式，名称，处理器）` | 同时登记反向名称。 |
| `取/发/更新/修补/删` | GET/POST/PUT/PATCH/DELETE 便捷入口。 |
| `命名取/命名发` | 命名 GET/POST。 |
| `路由组（前缀）` | 返回`言枢路由组`。 |
| `命名路由组（前缀，命名空间）` | 给组内名称加命名空间。 |
| `反向（名称，参数）` | 生成百分号编码 URL。 |

模式支持字面段、`:参数`和末尾`*通配`。`Web路由器`与`Web路由`公开供兼容和诊断使用；一般业务代码应经应用登记。

应用生命周期入口包括`使用`、`设错误处理器`、`设状态处理器`和`处理`。状态处理器只允许替换 404 或 405。

## 上下文

`言枢上下文`公开：

- 路径：`参数值`；
- 查询：`查询值/查询全部`；
- 请求：`首部值/Cookie值/正文文字/JSON正文/言据正文为/言据正文/是AJAX`；
- 状态：`状态值`；
- 会话：`绑定会话/会话`；
- 反向 URL：`绑定应用/反向`。

`Web上下文`继承`言枢上下文`，用于兼容旧类型名。`JSON正文`和`言据正文`都会先验证媒体类型；解析失败使用结构化错误前缀。

## 响应工厂

| API | 结果 |
| --- | --- |
| `HTML响应/HTML状态响应` | 安全渲染言页节点。 |
| `JSON响应` | JSON UTF-8 响应。 |
| `言据响应` | 已规范言据文字响应。 |
| `言据值响应` | 先规范序列化言据值。 |
| `协商媒体类型/协商响应` | 在 JSON 与言据间协商。 |
| `文字响应/字节响应/状态响应` | 基础正文响应。 |
| `重定向/永久重定向` | 302/301。 |
| `空响应` | 204。 |
| `Cookie` | 创建受验证的 HTTP Cookie。 |

模板响应入口包括模块级`言标响应/言标源码响应/言标状态响应`和应用级`模板响应/模板状态响应`。

## 会话

`会话存储协议`定义：

```yanxu
读取（编号：文）：典?
写入（编号：文，数据：典，最大秒：数）：空
删除（编号：文）：空
```

`会话存储适配器`返回`回调会话存储`。框架会在协议边界复制数据，但不会替存储实现并发控制或持久化。

`会话配置（）`返回`会话选项`，可配置 Cookie 名、路径、最大秒、数据预算、Secure、HttpOnly 和 SameSite。`会话中间件（选项，存储）`绑定会话。

`言枢会话`提供`编号/原编号/取/设/删值/所有/轮换/销毁/须保存/须轮换/已销毁`。销毁后不允许继续写入；轮换会使旧编号失效并写入新 Cookie。

辅助工厂为`会话Cookie`和`删除会话Cookie`。

## CSRF

`CSRF配置（）`返回`CSRF选项`。设置法覆盖：

- `加可信来源/设可信来源`；
- `设允许无来源/设要求令牌`；
- `设令牌首部名/设令牌Cookie名`；
- `设最大令牌字符`；
- `设安全/设仅HTTP/设同站`。

`CSRF令牌（）`生成令牌；`CSRF令牌Cookie/删除CSRF令牌Cookie`构造响应 Cookie；`CSRF中间件（配置）`执行来源和双提交校验。

## 静态文件

`静态配置（）`返回`静态选项`，设置法包括`设最大文件字节`、`设缓存控制`、`设允许隐藏文件`、`设条件请求`和`设范围请求`。

应用入口：

- `挂静态为（URL前缀，文件根，选项）`；
- `挂静态（URL前缀，文件根）`；
- `挂配置静态为（URL前缀，选项）`；
- `挂配置静态（URL前缀）`。

底层公开入口`静态文件响应为/静态文件响应/静态请求响应/静态处理器为/静态处理器`适合自定义组合。`静态请求响应`才能根据请求首部处理条件与范围语义。

## OpenAPI

`接口说明（操作标识）`返回`接口操作`。构建器提供：

- `设摘要/设说明/设标签/设弃用`；
- `设参数/设请求体/清请求体/设响应/设安全`；
- `设扩展`，名称必须以`x-`开头；
- `快照`。

应用显式登记入口为`接口路由`、`命名接口路由`、`接口取`和`接口发`。读取与输出入口为`接口快照`、`生成OpenAPI`、`OpenAPI响应`和`挂OpenAPI`。

`OpenAPI提供器协议`定义`生成（接口快照）：典`。`OpenAPI提供器（回调）`返回`回调OpenAPI提供器`。提供器失败归一化为`YANSHU_OPENAPI_PROVIDER`；大小超限为`YANSHU_OPENAPI_LIMIT`。

## 测试客户端

`测试客户端（应用）`返回`言枢测试客户端`：

- `请求（方法，路径）`；
- `请求为（方法，路径，首部）`；
- `请求字节为（方法，路径，首部，字节正文）`；
- `请求Cookie（方法，路径，Cookie典）`；
- `带正文/取/发/发JSON/删`。

任意首部和 Cookie 仍由`yanxu-http`严格解析。调用方不能注入客户端管理的分帧首部。

## `言访测试`子模块

```yanxu
引「包:web/言访测试」为 言访测试；
```

| API | 契约 |
| --- | --- |
| `传输（应用）` | 返回言访自定义受控传输。 |
| `客户端为（应用，基础地址）` | 返回绑定应用和指定基础地址的言访客户端。 |
| `客户端（应用）` | 基础地址为`http://testserver`。 |

传输不打开网络，支持受控重定向和重复响应首部。它不模拟真实网络能力。

## 言标

`言标`是`言标门面`单例，提供`环境/渲染/信任HTML`。`言枢模板环境`提供源码和文件渲染；模板根、扩展名、源码大小、包含深度和循环关系均受限制。详见[言标参考](yanbiao.md)。

## 开发服务器

`服务器（应用，地址）`返回`言枢开发服务器`；兼容入口`开发服务器`返回`Web开发服务器`。服务器提供`运行若干（最大请求）`和`运行`。它只用于回环开发与集成，不是生产 HTTP 宿主。
