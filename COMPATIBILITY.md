# 兼容性

## 版本矩阵

| 组件 | 当前锁定 | 兼容范围 | 用途 |
| --- | --- | --- | --- |
| 言序 | 1.1.12 | `>=1.1.12` | 类型检查、测试、机器 API 与构建 |
| yanxu-http | 1.0.0 | `^1.0` | HTTP/1.1 对象、解析和响应编码 |
| yanxu-html（言页） | 1.0.0 | `^1.0` | 安全 HTML 节点与渲染 |
| yanju（言据） | 1.2.0 | `^1.2` | 言据解析、规范化、限制和深复制 |
| yanxu-request（言访） | 1.0.0 | `^1.0` | 独立无端口测试传输 |

`言序.lock`固定上述版本的精确提交与内容校验和。发布包不得依赖临时分支。

## 公共 API 兼容

现代入口使用`言枢配置/言枢应用/言枢上下文/言枢开发服务器`。以下旧名保留：

- `应用（）`；
- `Web应用`；
- `Web上下文`；
- `开发服务器（应用，地址）`；
- `Web开发服务器`。

旧类型继承对应言枢类型。新代码应使用现代名称；兼容入口不会获得独立的第二套行为。

## 数据格式

- JSON 使用标准 JSON UTF-8；
- 言据使用`yanju 1.x`规范媒体类型和规范序列化；
- OpenAPI HTTP 输出使用`application/vnd.oai.openapi+json; version=3.1; charset=utf-8`；
- 会话数据只接受言据可深复制的基础值、列和典；
- HTTP 首部与 Cookie 由`yanxu-http 1.x`验证。

普通 JSON 不能冒充言据。OpenAPI 提供器返回典不代表文档已通过完整 OpenAPI Schema 校验。

## 平台

核心框架、路由、中间件、会话状态机、CSRF、OpenAPI 和测试适配器是纯言序实现，支持言序 1.1.12 可运行的平台。

文件模板与静态文件依赖宿主文件系统。开发服务器依赖标准套接字库，并只在清单许可的`127.0.0.1`、`::1`和`localhost`监听。仓库 CI 的稳定门禁以 Linux 为主；本地验收同时覆盖 macOS ARM64。

## HTTP 边界

底层请求对象遵循`yanxu-http 1.0`。言枢内建服务器只承诺：

- HTTP/1.1；
- 一个连接读取一个请求；
- 串行接受；
- 最终响应后关闭连接；
- 回环监听。

不承诺 TLS、HTTP/2/3、生产 keep-alive 工作池、流式上传、反向代理信任、Unix socket、零停机重启或多进程调度。

## 浏览器安全

默认安全首部、Secure/HttpOnly/SameSite Cookie 和 CSRF 中间件可在现代浏览器使用。应用仍需根据部署域名、HTTPS、嵌入策略和资源来源配置 CSP 与可信来源。

`__Host-`Cookie 需要 HTTPS、Secure、路径`/`且无 Domain。纯 HTTP 本地开发必须换用不带该前缀的名称后才能关闭 Secure。

## 言访适配器

`包:web/言访测试`要求言访 1.x 的自定义传输和响应模型。适配器支持：

- 查询、首部、Cookie 和字节正文；
- JSON 与言据；
- 受控重定向；
- 重复响应首部；
- HEAD 空正文语义。

它不支持真实 DNS、TLS、连接/读取/写入故障、套接字断开或网络时延。任意 HTTP(S) 主机都会映射到同一个内存应用。

## 升级策略

1.0 稳定线内，新增功能使用次版本，兼容修复使用补丁版本。删除兼容类型、改变错误代码、改变默认安全策略或修改公开参数属于破坏性变更，需要新的主版本并提供迁移文档。
