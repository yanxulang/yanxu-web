# 安全策略

## 支持版本

| 版本线 | 安全更新 |
| --- | --- |
| 当前稳定 1.x | 支持 |
| 0.2.x | 不支持 |
| 0.1.x | 不支持 |

使用者应运行最新稳定补丁版本，并保持言序、`yanxu-http`、`yanxu-html`、`yanju`和`yanxu-request`锁图在受支持版本。

## 私下报告

请通过 GitHub Security Advisory 私下报告漏洞。报告应包含：

- 受影响版本与提交；
- 最小复现；
- 预期与实际安全边界；
- 影响范围；
- 已知缓解方式。

不要在公开 issue、日志或复现仓库中放入真实 Token、Cookie、会话编号、个人数据、数据库连接串或生产主机信息。

## 响应范围

维护者会确认报告、复现问题、评估影响并准备修复。安全修复必须通过相关回归规格、完整规格、文档门禁和 Release 构建后发布。

## 重要边界

- 言标`信任HTML`不是清洗器；
- 会话存储协议不是内置生产存储；
- CSRF 不是认证或授权；
- OpenAPI 提供器输出不是自动通过 Schema 校验；
- 弱 ETag 不是内容完整性证明；
- 无端口测试传输不是 TLS、DNS 或真实网络测试；
- 内建开发服务器不应直接暴露到公网。

完整威胁模型和部署建议见[安全模型](docs/SECURITY_MODEL.md)。
